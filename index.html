CAD design of the suit .

The project is based on creating a system for the development of gross motor skills. This helps the user with spatial awareness, which is essential for development in adulthood, enabling them to understand their location and the objects around them.

Once the schematic was completed, the traces were routed to connect the components and the pins where the components need to be soldered. The boards were manufactured by JLCPCB..

After the boards were assembled, the cases were printed, and the programming was ready, the system’s straps were assembled.

implementation of the proyect

This project represents a significant advancement in the integration of hardware and software for medical applications, standing out for its ability to provide real-time physical feedback during specific motor development activities. The data obtained from both the control user and the test user should feed the model to create a more robust database for conducting a deeper study. Feedback from end users has been positive and focused on the continuous improvement of the system, thereby ensuring a user-centered approach and enhancing the quality of life for people with special needs.

Design of a delta-type robot.

Robot designe.

Front view

The main idea of this project was to design a delta-type robot from scratch, performing the necessary calculations for its movement, as well as developing the code for its operation and creating a mechanical system that would enable it to move along the X, Y, and Z axes.

Side view

The project had three main parts:

  • Design: The design phase was carried out using SolidWorks software, where the mechanical system enabling the movement was designed, utilizing lead screws. Additionally, the structure that housed the movement systems was also designed.
  • Electronics: In the electronics, an Arduino-type programming board was used to control three NEMA stepper motors, as well as three limit switches to prevent the arms from colliding with the structure. Additionally, a board with three H-bridges was used for controlling the NEMA motors.
  • Programming: It was programmed in the Arduino IDE to control the movements of the robot along the X, Y, and Z axes.

Mechanical tool for Fanuc robot.

Robotics and Tooling.

The purpose of this project was to develop a tool for the Fanuc robot to simulate a production line where the window is placed on the car’s medallion. In this project, in addition to creating the tool, the robot programming was developed to follow this routine in collaboration with another robot from the Kuka brand. These robots work collaboratively via a PLC, where, once the window placement routine (with the Kuka) is completed, the glue placement routine begins.

The design of the tool worked correctly, as did the programming of the PLC and the Kuka robot. They collaborated seamlessly for the simulation of the production process of windshield placement.

Development of a Mobile Robot

Development of an movil robot with UX and UI

The objective of this project was to design a mobile omnidirectional robot that could be controlled through an application running on a Raspberry Pi. With the application, it was possible to control the direction of movement as well as the power of the motors.

The design of the interface was created using Figma software. This application was controlled from a Raspberry Pi via Wi-Fi communication, where the car, through an Arduino, connected to a Wi-Fi network, just like the Raspberry Pi. Once both devices were connected to the same network, the car’s control was executed, allowing the user to move the car as desired.

Design of a monitoring and early warning system for sudden cardiac death in older adults

Health Care..

Design a monitoring system for the study and early warning of sudden cardiac death episodes in older adults. The specific objectives include interviewing a medical expert to identify the symptoms and existing devices related to sudden cardiac death. This interview will provide valuable insights into the current knowledge and tools available in the field. Following this, the information gathered will be used to create a comprehensive list of the system’s requirements and features. This will ensure that the system is designed with the necessary functionality to effectively monitor and alert for potential cardiac events. Additionally, the system’s architecture will be established, determining how its various components will interact and function together. This will lay the groundwork for both the hardware and software aspects of the monitoring system. Lastly, an optimal communication method between the software and hardware of the monitoring system will be identified. This step will ensure seamless integration and effective data transfer, essential for the real-time monitoring of the patient’s condition.

We proposed a system based on a circuit embedded in a compression shirt, which would include three devices: an electrocardiogram, a pulse oximeter, and a body thermometer. The system’s microcontroller would be an ESP32, which would send the data to a server for processing.

The data processing would follow this route: if the obtained data meets the alert parameters, it would communicate with the attending physician and a family member for prompt attention.

The system aims to improve the quality of medical treatment by remotely monitoring patients’ health, as current equipment for such studies is often uncomfortable and inconvenient for patients.
